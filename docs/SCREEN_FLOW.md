## 1. 画面一覧

| 画面ID | 画面名 | 用途 |
|---|---|---|
| S-02 | 商談入力画面 | 商談内容のテキスト入力・音声入力・店舗選び・送信 |
| S-03 | 保存完了画面 | 保存結果の表示・連続入力への導線 |

S-01（初回設定画面）は Googleログイン方式への変更により **廃止**。担当者・営業所はログイン中の Google アカウントのメールアドレスから自動特定する。

本アプリは SPA 方式で構成する。エントリ `doGet` は初回アクセス時のみ呼ばれ、S-02 と S-03 の切替はクライアント側の DOM 操作で行う（サーバーへのページ遷移は行わない）。URLパラメータは使用しない。

## 2. 全体の遷移

```
共通URLにアクセス
  → doGet がHTMLを返す
  → GAS が Session.getActiveUser().getEmail() でログイン者を取得し、担当者マスタと照合（validateUser）
      ├─ マスタに無い／無効フラグ → エラー表示「登録されていない担当者です。管理者に連絡してください」（S-02に進まない）
      └─ 有効 → クライアントが localStorage の未送信テキスト（unsent_text）を確認
              ├─ 無し → S-02 を表示（入力欄は空。担当者名を画面上部に表示）
              └─ 有り → S-02 を表示（入力欄に復元。「前回の入力が未送信です。送信してください」を上部に通知）

S-02 で入力（音声またはテキスト）→ 送信
  → localStorage に unsent_text を保存 →「記録しています…」表示
  → 送信時に位置情報を取得 → 近い順4店舗を候補表示 → タップ選択
        （該当なし → 店名を手入力 → 部分一致で名寄せ候補 → 選択）
  → submitMemo（サーバーで担当者特定・AI解析・分割・書き込み）
      ├─ 成功 → localStorage の unsent_text を削除 → S-03 を表示
      └─ 失敗（AI API等）→ エラー表示。入力欄はクリアしない。unsent_text も保持
                          → 再アクセス時に入力欄へ復元（上記の「有り」の流れ）

S-03 で「続けて入力する」→ S-02 を表示（入力欄クリア済み）
```

## 3. S-02（商談入力画面）の構成

- 担当者名の表示（ログインで特定した担当者名を画面上部に表示）
- マイクボタン（音声入力・主手段）：画面上で優先配置する。押下で音声認識開始、認識中を視覚表示、再押下または無音で停止。認識結果は入力欄の末尾に追記（上書きしない）。Safari では自動停止する場合があり、再押下で追記する
- テキスト入力欄（補助手段）：キーボード入力に加え、通話文字起こしテキストの貼り付けにも使う
- 送信ボタン：押下で送信処理を開始
- 店舗候補リスト（近い順4店舗のタップ選択）
- 店舗名手入力欄（候補に該当なしの場合のフォールバック。部分一致で名寄せ候補を出す）
- 未送信テキストの通知（localStorage に未送信が残る場合に表示し、入力欄へ復元）

音声入力は、ブラウザ仕様上タップなしの自動録音は行わない（「主手段」は UI 上の優先度・視覚的な主従を指す）。S-02 のレイアウト詳細（マイクボタンと入力欄・店舗選び UI の具体配置）は未確定であり、画面遷移図・画面一覧と対象工程の実装指示書を正とする。

---

## 4. S-03（保存完了画面）の構成

- 分割されたレコード数の表示（例「記録しました（3件に分割）」）
- 各レコードの概要（顧客名・案件テーマ・宿題の有無）
- 「続けて入力する」ボタン（S-02 へ戻る。入力欄はクリア済み）

抽出結果のプレビュー・営業による確認ステップは設けない（バック処理で完結する）。

---

## 5. 異常系の扱い

- 担当者マスタ不一致・無効：エラー表示のみ。S-02 に進めない
- 位置情報の取得拒否・該当店舗なし：緯度・経度・店舗名は空欄のまま記録を続ける（エラーで止めない）
- AI API 障害・ネットワーク障害：エラーメッセージを表示し、入力欄をクリアしない。localStorage の未送信テキストも削除しない。次回アクセス時に復元する
- 音声認識不可（Web Speech API 障害）：テキスト入力で代替できるようガイドを表示
