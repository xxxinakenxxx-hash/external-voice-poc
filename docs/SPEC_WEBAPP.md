## 2. Webアプリの役割

この Webアプリ（Phase 1）は、次を担当する。

- 営業担当による商談内容の入力（音声を主手段、テキストを補助手段）
- Googleログインによる担当者・営業所の自動特定
- 入力テキストの AI解析・構造化（顧客 × 案件テーマ単位への分割）
- 構造化レコードのスプレッドシート蓄積
- 送信時の位置情報取得と、店舗マスタ照合による店舗選び
- 抽出失敗の可視化（空欄＋赤背景）と、エラーの記録

活用機能（検索・ブリーフィング・レポート・配信等）は Phase 2 であり、本アプリの対象外。

## 3. 利用者

- 営業担当（本社6名・パイロット）：商談記録の入力・送信、保存結果の確認。スプレッドシートは直接触らない。
- 情シス管理者（1名）：スプレッドシートの閲覧・編集、担当者マスタ・店舗マスタの管理、赤背景行の週次確認・修正、GAS修正・デプロイ。

## 6. 主要な業務ルール

### 6.1 入力

入力の主手段は音声（マイクボタン）、補助手段はテキストとする。画面はマイクボタンを優先配置する。ブラウザ仕様上、タップなしの自動録音は行わない（「主手段」はUI上の優先度を指す）。音声認識の安定性は端末に依存するため、実測次第で音声・テキストを並列構成に変更する場合がある。

入力は自由文でよく、複数顧客・複数案件の混在を許容する。通話の文字起こしテキストの貼り付けにも同じ入力欄で対応する（追加処理なし）。

### 6.2 未送信テキストの保全

送信ボタン押下時に入力テキストをブラウザの localStorage（キー名 unsent_text）へ一時保存し、書き込み正常完了後に削除する。次回アクセス時に未送信テキストが残っていれば入力欄へ復元し、画面上部に通知を出す。localStorage の用途はこの一時保全に限る（担当者情報は保存しない）。

### 6.3 担当者識別

Googleログイン（会社の Google Workspace アカウント）を必須とする。GAS がログイン者のメールアドレスを取得し（Session.getActiveUser().getEmail()）、それをキー（user_key＝メールアドレス）に担当者マスタと照合して担当者名・営業所名を引き当てる。初回設定画面（S-01）は廃止。担当者の切替操作は設けない。

マスタに存在しない、または無効フラグの場合は「登録されていない担当者です。管理者に連絡してください」と表示し、入力画面へ進めない。

### 6.4 AI解析・構造化

入力テキストを OpenAI API に送信し、構造化 JSON に変換する。Phase 1 の AI 使用箇所はこの構造化・分割のみで、送信ごとに1回呼び出す。宿題期限は入力テキストの表現（「急ぎ」「来月」「3/25」等）をそのまま記録し、実日付への変換は行わない。

### 6.5 レコード分割（最重要）

AI は入力テキストを「顧客 × 案件テーマ」単位でレコードに分割する。

- 複数顧客が混在 → 顧客ごとに分割
- 同一顧客で複数の案件テーマ → 案件テーマごとに分割
- 案件テーマが1つ → 1レコード

案件テーマの粒度は「商談の話題単位」とし、個別の商品名・品種・SKU 単位では分割しない（例：「油脂（マーガリン・ショートニング）」は1レコード、「油脂」と「ジェラートマシン」は2レコード）。分割ルールと粒度基準は AI の System Prompt で指示する（具体文は AIプロンプト設計書で策定）。

### 6.6 データ格納

スプレッドシートに1行1レコードで格納する。常に新規行として追加し、過去レコードの上書き・更新はしない。修正が必要な場合は管理者がスプレッドシートを直接編集する。

### 6.7 抽出失敗時の挙動

AI が抽出できなかった項目は空欄のまま書き込み、処理を止めない。必須抽出項目（顧客名・案件テーマ・商談内容）が空欄の場合、または分割自体が不能だった場合は抽出ステータスを「要確認」とし、空欄の必須セル（顧客名・案件テーマ・商談内容の列）背景を赤（#FF0000）にする。宿題が無い場合の空欄は正常であり失敗ではない。管理者が週次で赤背景行を確認・修正する。

### 6.8 位置情報・店舗選び

送信時にブラウザの Geolocation API で緯度・経度を取得する。取得した座標を店舗マスタと距離照合し、近い順4店舗を候補表示してタップ選択させる。該当がなければ店名を手入力し、部分一致で名寄せ候補を出して選ぶ。確定した店名と座標を商談記録に格納する。取得拒否・該当なしの場合は空欄のまま記録を続ける（エラーで止めない）。外部APIによる住所変換は使わない。

### 6.9 保存完了の表示

保存後、分割されたレコード数と各レコードの概要（顧客名・案件テーマ・宿題の有無）を表示する。「続けて入力する」で入力画面へ戻る（入力欄はクリア済み）。抽出結果のプレビュー確認ステップは設けない。
