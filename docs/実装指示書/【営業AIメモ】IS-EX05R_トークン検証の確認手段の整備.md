## 2. 本工程の目的

トークン検証（verifyToken）が、正しいトークンを通し、**改ざん・期限切れ・トークンなしを確実に拒否する**ことを、けんたろうが画面操作だけで確認できる状態にする。

**なぜここを厳格にやるか：** 送信API（デプロイB）はアクセス範囲「全員」で公開されている。URLさえ分かれば誰でも叩ける。トークン検証が、なりすまし防止の唯一の防御線である。ここが機能しないままEX-06でAI解析・スプレッドシート書き込みに接続すると、他人になりすました商談記録が本番シートに書き込める状態になる。**確認できるまでEX-06へ進まない。**

## 4. Codexの作業内容

### 4-1. `external/voice-poc/index.html` に確認専用の入力欄を追加する

現行の「GAS通信検証」欄を、次の構成に置き換える。**音声入力部分（EX-01の実装）は変更しない。**

| 要素 | 内容 |
|---|---|
| GAS送信先URL入力欄 | 現行のまま維持。送信API用デプロイBのURLを貼り付ける |
| **トークン入力欄（新規）** | 複数行のtextarea。トークンは長いため1行だと編集しづらい |
| **inputText入力欄（新規）** | 1行のテキスト入力で足りる。既定値は `EX-05R test` とする |
| 送信ボタン | 現行の「GAS通信テスト」ボタンを流用。押下時にURL・トークン・inputTextの3つを送信する |
| 応答表示エリア | 現行のまま維持。HTTPステータスと応答本文（JSON）をそのまま表示する |

**トークンの自動反映：** ページ読み込み時にURLのフラグメント（`#token=...`）があれば、トークン入力欄に自動で流し込む。TOP画面から遷移してきたときに、貼り付け作業なしで正しいトークンが入る状態にする。

**手動編集・全削除：** 自動反映後も、トークン入力欄は自由に編集・全削除できること。改ざん確認（1文字書き換え）と、トークンなし確認（空欄）を画面操作だけで行うためである。

**送信内容：** `{ "token": <トークン入力欄の値>, "inputText": <inputText入力欄の値> }` をJSON文字列にしてPOSTする。トークン入力欄が空の場合は、`token` を空文字として送る（送信自体をブロックしない。**空トークンがGAS側で拒否されることを確認するのが目的**のため）。

**Content-Type：** `text/plain;charset=UTF-8` を維持する。**変更しない**（EX-02で成立したCORS回避の形式であるため）。

**固定値の撤去：** `EX02_DUMMY_TOKEN` の固定送信は廃止する。

### 4-2. `src/` に検証専用の短命トークン発行関数を追加する

有効期限は12時間のため、期限切れの確認を実機で待つのは非現実的である。そこで、GASエディタから直接実行する検証専用の関数を追加する。

- 関数名：**`issueShortLivedTokenForTest`（末尾のアンダースコアを付けない）**
- 内容：`issueToken` と同じ署名方式・同じpayload仕様で、**有効期限を60秒**にしたトークンを発行し、`Logger.log` に出力する。対象のメールアドレスは、けんたろう自身のメールアドレスを関数内の定数で指定する。
- **GASエディタの関数選択欄から手動実行するため、末尾にアンダースコアを付けてはならない。** Apps Scriptでは、末尾が `_` の関数は private 扱いとなり、エディタの関数選択欄に表示されず、手動実行できない。
- **`doGet`・`doPost`・`issueToken`・`verifyToken`・`getTokenSecret` からは呼ばない。** Webアプリの通常処理からは到達させない。GASエディタからの手動実行専用とする。
- 署名方式・payload仕様（`{uh, exp}`＋HMAC-SHA256署名）は `issueToken` と同一とする。**変更しない。** 変えると、この関数で発行したトークンでの確認が、本番のトークンの確認にならない。
- 有効期限のみ60秒とする。

**この関数はEX-06完了後に削除する。** README等にその旨を明記する。

**補足（Webアプリからの到達可能性について）：** 末尾のアンダースコアを外すことで、この関数はエディタから実行可能になる。ただしGAS Webアプリで外部から呼べるのは `doGet` / `doPost` のエントリポイントのみであり、任意の関数名を外部から直接叩くことはできない（`google.script.run` を使うHtmlServiceの画面からのみ任意関数を呼べるが、本システムのTOP画面はトークン発行のみを行い、この関数を呼ばない）。したがって、末尾の `_` を外しても外部から悪用される経路は生じない。

## 9. けんたろうへの作業依頼

Codexの作業完了後、順に実施する。**手順4・5・6が本工程の核心である。省略しない。**

**手順1：TOP画面（デプロイA）を開く。**
「入力画面へ」のリンク先URLをコピーする（本番の入力ページのURLに `#token=...` が付いたもの）。

**手順2：検証用ページにトークンを持ち込む。**
`external/voice-poc/` の検証用ページを開き、コピーしたURLの `#token=` 以降のトークン文字列を、トークン入力欄に貼り付ける。GAS送信先URL欄には、送信API用デプロイBのURLを入れる。

**手順3（正しいトークン）：そのまま送信する。**
→ **期待：HTTP 200 が返り、応答に自分のメールアドレス（担当者が確定した結果）が含まれる。**

**手順4（改ざんトークン）：トークン入力欄の文字を1文字だけ書き換えて送信する。**
（例：末尾の1文字を別の英数字に変える）
→ **期待：拒否される。メールアドレスが返らない。「セッションが無効です」等のエラーが返る。**

**手順5（トークンなし）：トークン入力欄を全部消して、空欄のまま送信する。**
→ **期待：拒否される。メールアドレスが返らない。**

**手順6（期限切れトークン）：**
1. GASエディタを開き、上部の関数選択欄から `issueShortLivedTokenForTest` を選び、実行する。（※末尾にアンダースコアが付いていると関数選択欄に表示されない。表示されない場合はCodexの修正漏れであり、報告すること）
2. 実行ログに出力された短命トークン（有効期限60秒）をコピーする。
3. 検証用ページのトークン入力欄に貼り、**すぐに送信する** → **期待：HTTP 200。メールアドレスが返る**（＝有効期限内なので通る）。
4. **そのまま2分ほど待ってから、同じトークンでもう一度送信する** → **期待：拒否される。メールアドレスが返らない**（＝期限切れで弾かれる）。

**手順7：結果を報告する。**

## 11. 停止条件と戻り先

**停止条件：** 改ざん・空欄・期限切れのいずれかが**拒否されず、メールアドレスが返ってしまう**。

**戻り先：** これは確認手段の問題ではなく、**トークン検証の欠陥である。** 直ちに停止し、`verifyToken` の該当箇所（署名の再計算・照合／有効期限の比較／user_hashの突き合わせ）を切り分けて修正する。修正後、本工程の手順3〜6を最初からやり直す。

**絶対に守ること：** 拒否されない状態でEX-06へ進んではならない。EX-06はAI解析とスプレッドシート書き込みに接続する工程である。防御線が機能しないまま接続すると、誰でも他人になりすまして本番の商談記録を書き込める状態になる。
