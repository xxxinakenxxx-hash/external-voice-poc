# EX-04：外部入力専用マイクページ本体の作成

## 0. 最初に必ず読むこと

1. `tasks/TASK_CURRENT.md`
2. `docs/ルールブック.md`
3. `AGENTS.md`
4. `external/voice-poc/index.html`（EX-01の音声入力実装。流用元として参照する）

## 1. 背景

GAS HtmlServiceのiframe内ではマイク権限が渡されず、音声入力が動作しないことが判明した。そのため、入力画面のみを外部の静的ホスティングに出す構成（案B改）に変更した。この構成の成立は検証済みである。

本工程では、その**本番用の外部入力ページ**を新規作成する。GASへの接続はまだ行わない。

## 2. 今回の作業

`external/input-page/` を新規作成し、次の2ファイルを作る。

- `external/input-page/index.html`（HTML・CSS・JavaScriptを1ファイルにまとめる）
- `external/input-page/README.md`（ページの役割・デプロイ先・触ってよい範囲）

## 3. 実装するUI要素

| 要素 | 内容 |
|---|---|
| マイクボタン | 画面上部・目立つ位置。最小44px×44px。音声入力が主手段のため優先配置する |
| 録音中表示 | マイクボタンの色変化、または「録音中…」テキスト |
| テキスト入力欄 | 複数行のtextarea。音声認識結果が追記される。手入力・編集も可能 |
| 送信ボタン | 押下で送信処理を開始する |
| 処理中表示 | 「記録しています…」を表示し、送信ボタンを二重押下できないようにする |
| 結果表示エリア | 送信結果を表示する領域（本工程では未接続のためダミー表示） |
| 未送信通知エリア | 「前回の入力が未送信です。送信してください」を表示する |

## 4. 音声入力（Web Speech API）

- `window.SpeechRecognition || window.webkitSpeechRecognition` を使う。
- 言語は日本語（`ja-JP`）。
- **認識結果はテキスト入力欄の末尾に追記する。既存テキストを上書きしない。**
- マイクボタン再押下、または無音での自動停止に対応する。
- iPhone Safariでは約60秒で自動停止する場合がある。停止後、再度マイクボタンを押せば追記を再開できるようにする。
- Web Speech APIが利用できないブラウザでは、マイクボタンを無効化し、テキスト入力で継続できるようにする。

## 5. 未送信テキストの保全

- 送信ボタン押下時に、入力テキストを localStorage に保存する（キー名：`unsent_text`）。
- ページ読み込み時に `unsent_text` が残っていれば、テキスト入力欄に自動復元し、未送信通知を表示する。
- **本工程では削除処理は実装しない**（削除はEX-06で、書き込み成功後に行う）。削除を呼ぶ関数の枠だけ用意しておく。

## 6. 送信処理（本工程では未接続）

送信ボタン押下時、次までを実装して止める。

1. 入力テキストが空でないことを検証する（空なら送信しない）。
2. `unsent_text` を localStorage に保存する。
3. 「記録しています…」を表示する。
4. **POSTは行わない。**「（EX-06で接続予定）」とダミー表示して処理を終える。

POST先URL・トークンの取り扱いは、本工程では一切実装しない。

## 7. デザイン

- モバイルファースト。スマートフォン縦画面を基準とする。
- 片手操作を前提。ボタンは最小44px×44px。
- カラー：`#1a3a5c`（メイン）、`#e8913a`（アクセント）、`#f5f5f5`（背景）、`#fff`。
- フォント：端末標準（sans-serif）。

## 8. やってはいけないこと

- **担当者名・メールアドレス・営業所名を、画面に表示しない。ソースに書かない。**
- user_master・branch_master・store_master の情報を持ち込まない。
- 過去の商談記録を表示しない。
- スプレッドシートID・OpenAI APIキーをソースに書かない。
- **外部JSライブラリ・CDN・外部フォントを一切使わない**（すべて自前のコードで書く）。
- アクセス解析タグ（Google Analytics等）を入れない。
- 商談本文をURLパラメータに載せない。
- 商談本文を外部サーバーへ送信・保存しない（本工程ではPOST自体を行わない）。
- `src/` 配下を変更しない。
- `external/voice-poc/`・`external/gas-poc/` を変更・削除しない。
- 正本文書（`docs/` 配下）を変更しない。
- 位置情報（Geolocation API）を実装しない。
- トークンの受領・保持・送信を実装しない。
- 次工程（EX-05以降）の作業を前倒ししない。

2. HTMLに外部リソース（`<script src=...>`・`<link href=...>`）が含まれていないことを grep で確認する。
3. HTMLに担当者情報・APIキー・スプレッドシートIDに相当する文字列が含まれていないことを確認する。
