## 3. やってよいこと

### 3-1. ページ構成（UI要素）

| 要素 | 内容 |
|---|---|
| マイクボタン | 画面上部・目立つ位置に配置する（要件定義FR-01：音声入力が主手段）。最小44px×44px。 |
| 録音中表示 | マイクボタンの色変化、または「録音中…」テキストで、認識中であることを示す。 |
| テキスト入力欄 | 複数行のtextarea。音声認識結果が追記される。手入力・編集も可能。 |
| 送信ボタン | 押下で送信処理を開始する。 |
| 処理中表示 | 送信中は「記録しています…」を表示し、送信ボタンを二重押下できないようにする。 |
| 結果表示エリア | 送信結果（成功／失敗）を表示する領域。本工程では未接続のため、ダミー表示で構わない。 |
| 未送信通知エリア | 未送信テキストが復元されたときに「前回の入力が未送信です。送信してください」を表示する。 |

### 3-2. 音声入力（Web Speech API）

- `window.SpeechRecognition || window.webkitSpeechRecognition` を使う。
- 言語は日本語（`ja-JP`）。
- **認識結果はテキスト入力欄の末尾に追記する。既存テキストを上書きしない**（要件定義FR-02：追記モード）。
- マイクボタン再押下、または無音での自動停止に対応する。
- iPhone Safariでは約60秒で自動停止する場合がある。停止後、再度マイクボタンを押せば追記を再開できるようにする。
- Web Speech APIが利用できないブラウザでは、マイクボタンを無効化し、テキスト入力で継続できるようにする（要件定義9-2：Web Speech API障害時はテキスト入力で代替）。

### 3-3. 未送信テキストの保全（要件定義FR-01）

- 送信ボタン押下時に、入力テキストを localStorage に保存する（キー名：`unsent_text`）。
- ページ読み込み時に `unsent_text` が残っていれば、テキスト入力欄に自動復元し、未送信通知を表示する。
- **本工程では、送信が成功しないため `unsent_text` の削除処理は実装しない**（削除はEX-06で、書き込み成功後に行う）。ただし削除処理を呼ぶ関数の枠だけは用意しておく。

### 3-4. 送信処理（本工程では未接続）

- 送信ボタン押下時の処理は、次までを実装する。
  1. 入力テキストが空でないことを検証する（空なら送信しない）。
  2. `unsent_text` を localStorage に保存する。
  3. 「記録しています…」を表示する。
  4. **ここで止める。** POSTは行わない。「（EX-06で接続予定）」とダミー表示して処理を終える。
- POST先URL・トークンの取り扱いは、本工程では実装しない。

### 3-5. デザイン

- モバイルファースト。スマートフォン縦画面を基準とする。
- 片手操作を前提。ボタンは親指で押せるサイズ（最小44px×44px）。
- カラーは要件定義8-6の暫定値（`#1a3a5c`, `#e8913a`, `#f5f5f5`, `#fff`）。
- フォントは端末標準（sans-serif）。
