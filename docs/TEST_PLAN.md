## 2. 機能別テスト観点

### 2.1 入力・送信（FR-01）
- テキスト入力→送信→記録まで通るか
- 送信途中でブラウザを閉じても、次回アクセスで未送信テキストが復元され、通知が出るか
- 通話文字起こしの貼り付けテキストが同じ入力欄で送信できるか

### 2.2 音声入力（FR-02）
- マイクボタンで認識開始、認識中の表示、再押下・無音での停止
- 認識結果が入力欄の末尾に追記される（上書きしない）
- Safari で自動停止後、再押下で追記できるか

### 2.3 担当者識別（FR-03）
- 有効な担当者でログイン→担当者名・営業所が特定される
- マスタ未登録・無効フラグ→エラー表示、入力画面へ進めない

### 2.4 AI解析・分割（FR-04・FR-05・FR-10）
- 同一店舗内で部門・相手が異なる→別レコードに分割される
- 同一店舗で複数テーマ→テーマごとに分割
- 分割後の全レコードに、画面で指定した同じ店舗（顧客名・商談先店舗ID）が入る
- 音声中に別の店舗名・部門名・運営会社名が出ても、指定店舗が変わらない
- AI出力に customer_name キーが含まれない（5キーであること）
- 話題単位の粒度（品種・SKUで過分割しない）
- 宿題期限が原文表現のまま記録される（日付変換しない）

### 2.5 書き込み・抽出失敗（FR-06・FR-08）
- 各項目が正しい列に入るか、record_id が3桁連番か
- 必須項目（案件テーマ・商談内容）が空欄の行が「要確認」＋該当セル（G列・H列）赤背景になるか
- 顧客名（E列）が赤背景の対象になっていないか
- 宿題なしの空欄が誤って赤背景にならないか
- U列（store_name_gps）に新規の書き込みが発生していないか
- V〜Z列（商談先店舗ID・入力店舗名・訪問店舗ID・商談区分・記録者キー）が正しく入るか

### 2.6 商談区分・店舗指定・位置情報（FR-09・FR-10・FR-11）
- 画面を開いた直後に位置情報の許可ダイアログが出ないか
- 商談区分の初期選択が「訪問」になっているか
- 訪問選択時に「現在地から店舗を探す」を押すと近い順4店舗が出るか
- 該当なしで店舗名検索→部分一致候補が出るか
- 検索でも該当なしの場合、入力した店舗名がそのまま店舗指定として確定するか
- 店舗マスタから選んだ場合、保存された顧客名がマスタの正式名称になっているか（クライアントから送った文字列がそのまま入っていないか）
- 緯度・経度が S・T 列に入るか（訪問時のみ）
- 電話選択時に「現在地から店舗を探す」が表示されないか。緯度・経度が保存されないか
- 商談区分を切り替えたときに、店舗選択と位置情報が破棄され、選び直しを促す表示が出るか
- 位置取得拒否時にエラーで止まらず、店舗名検索へ誘導されるか（店舗指定は必須のまま）
- 店舗未指定・空白のみの店舗名で送信を試みたときに拒否され、次の操作が分かるメッセージが出るか
- 存在しない店舗ID・無効店舗を送ったときにGAS側で拒否されるか（画面側の検証だけに依存していないか）

### 2.7 保存完了（FR-07）
- 分割件数と各件概要が表示されるか
- 「続けて入力する」で入力欄が空の状態に戻るか

- 個人日報（S-05）に自分の記録だけが表示されるか
- S-05で対象記録を選択すると追記画面（S-04）が開くか。追記対象がrecord_idで特定されているか

## 4. 異常系の確認観点

AI異常は次の5つを区別して確認する。

| # | 異常の種類 | 期待結果 |
|---|---|---|
| 1 | AI APIのタイムアウト | 商談記録が保存されない。「現在利用できません。しばらく経ってから、もう一度送信してください。」が表示される。入力欄はクリアされず、未送信テキストが保持・復元される |
| 2 | AI APIのHTTP 4xx（認証エラー・リクエスト不正等） | 商談記録が保存されない。「現在利用できません。管理者に連絡してください。入力内容は保持しています。」が表示される。未送信テキストが保持される。error_logへ記録される |
| 3 | AI APIのHTTP 5xx（AI事業者側の一時障害） | 商談記録が保存されない。「現在利用できません。しばらく経ってから、もう一度送信してください。」が表示される。未送信テキストが保持される。error_logへ記録される |
| 4 | AIから応答は得たがJSON解析不能 | **このときだけ**全文が1レコードとして保存される。extract_statusが「要確認」、G列・H列が赤背景になる。確定した店舗情報（E・V・W・X・Y列）とZ列も正常に保存される。未送信テキストは削除される |
| 5 | スプレッドシートへの書込失敗 | 保存失敗として扱われる。「保存に失敗しました。しばらく経ってから、もう一度送信してください。」が表示される。未送信テキストが保持される |

- ネットワーク障害（送信APIへ到達できない）：エラー表示、入力欄クリアされない、未送信テキスト保持・復元
- 担当者マスタ不一致・無効：エラー表示のみ
- 位置情報の取得拒否：送信エラーとせず、「現在地を取得できませんでした。店舗名で検索してください」と表示して検索へ誘導する。緯度・経度は空欄のまま。店舗指定は必須であり省略できない
- 店舗未指定・空白のみの店舗名：送信を拒否し、次の操作が分かるメッセージを表示する
- 存在しない店舗ID・無効店舗：送信を拒否する（AI APIを呼ばない）
- 商談区分が未指定・不正：送信を拒否する
- 電話商談で緯度・経度が届いた場合：拒否せず、緯度・経度だけ破棄して処理を継続する
- AI解析失敗時：確定した店舗情報（顧客名・商談先店舗ID・入力店舗名・商談区分）は正常に保存される

```
- 対象：FR-xx / 画面 S-xx / 観点
- 手順：
- 期待：
- 結果：OK / NG / 違和感あり
- 備考（NGの場合は再現手順）：
```

NG・違和感は実装判断をせず、フィードバック一覧.md に記録する（ルールブック §19）。
